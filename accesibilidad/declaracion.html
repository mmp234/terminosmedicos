Situación de cumplimiento

Aprendizaje sobre términos médicos se compromete a proporcionar un sitio web accesible para todos los usuarios, independientemente de su discapacidad. Nos esforzamos por cumplir con las WCAG 2.2, publicadas por, en inglés, el W3C, siendo nuestro sitio web parcialmente conforme con el nivel AA.

Este sitio web es parcialmente conforme con la Directiva (UE) 2016/2102 debido a las excepciones y la falta de conformidad indicadas a continuación:

Contenido no accesible

Este sitio web presenta diferentes contenidos no accesibles, los cuales son:

  • Documento de Caso Clínico (Epilepsia -> Recursos)
  • Vídeos enlazados de Youtube que no permiten el control (Migraña -> Descripción, Artrosis -> Descripción)
  • Blog insertado de WordPress que no permite modificaciones ni control sobre este (Página de inicio)

Procedimiento de aplicación

Si no quedas satisfecho con nuestra respuesta, puedes dirigirte a la Unidad de Accesibilidad dependiente del Ministerio de Derechos Sociales y Agenda 2030, o bien realizar una solicitud de queja formal a través del Procedimiento de aplicación.

Identificación del idioma

El idioma principal de esta página web es el español, indicado mediante el atributo lang="es" en la etiqueta <html>. Los cambios de idioma dentro del contenido se identifican utilizando el atributo lang en las etiquetas correspondientes. (WCAG 2.2, Criterio 3.1.1 - Nivel A, Criterio 3.1.2 - Nivel AA)

Etiquetado semántico

Utilizamos etiquetas semánticas de HTML5 para estructurar el contenido de la página de forma significativa, incluyendo <header>, <nav>, <main>, <article>, <section>, <footer>, etc. Las etiquetas de encabezado (<h1> a <h6>) se utilizan para estructurar la página con títulos y subtítulos, siguiendo un orden jerárquico. (WCAG 2.2, Criterio 1.3.1 - Nivel A)

Texto alternativo para imágenes

Todas las imágenes de este sitio web tienen un texto alternativo (atributo alt) que describe su función. Las imágenes decorativas tienen un atributo alt vacío (alt=""). (WCAG 2.2, Criterio 1.1.1 - Nivel A)

Mecanismos de navegación

Este sitio web proporciona un mapa de navegación y una barra de posición al inicio de cada página para facilitar la navegación. Los textos de los enlaces son claros y descriptivos, lo que permite a los usuarios comprender su propósito. (WCAG 2.2, Criterio 2.4.5 - Nivel AA, Criterio 2.4.8 - Nivel AAA)

Enlaces

El texto de los enlaces es significativo, descriptivo y comprensible fuera de contexto. Se evita el uso de "clic aquí" o frases genéricas. Cada enlace tiene un texto único. (WCAG 2.2, Criterio 2.4.4 - Nivel A)

Formularios

Las instrucciones del formulario son claras y concisas, se indican los campos obligatorios con un asterisco y se explican los formatos de entrada.

Se utiliza label con el atributo for para asociar etiquetas a controles.

Se agrupan controles relacionados con fieldset y lengend, especialmente botones de opción y casillas de verificación.

Se verifica el orden de tabulación para asegurar una navegación lógica con el teclado.

(WCAG 2.2, Criterio 3.3.2 - Nivel A)

Títulos

Cada página tiene un título único y descriptivo definido con la etiqueta title, incluyendo el título de la página y el título del sitio web. (WCAG 2.2, Criterio 2.4.2 - Nivel A)

Diseño visual y CSS

Además de las medidas de accesibilidad implementadas con HTML, este sitio web utiliza CSS para mejorar la experiencia de usuario y la accesibilidad en las siguientes áreas:

  • Contraste: Se ha asegurado un contraste de color adecuado entre el texto y el fondo para facilitar la lectura. Se proporciona un estilo de alto contraste para usuarios con baja visión. (WCAG 2.2, Criterio 1.4.3 - Nivel AA, Criterio 1.4.6 - Nivel AAA)
  • Tamaño del texto: El tamaño del texto es ajustable mediante el zoom del navegador y se proporciona un estilo con un tamaño de letra más grande para facilitar la lectura. (WCAG 2.2, Criterio 1.4.4 - Nivel AA)
  • Espaciado: Se ha añadido un espaciado adecuado entre párrafos, elementos de lista y otros elementos para mejorar la legibilidad. (WCAG 2.2, Criterio 1.4.12 - Nivel AA)
  • Tipografía: Se han elegido tipos de letra claros y legibles para mejorar la experiencia de lectura. (WCAG 2.2, Criterio 1.4.8 - Nivel AAA)
  • Anchura del texto: Se ha limitado la anchura del texto para facilitar la lectura en pantallas grandes. (WCAG 2.2, Criterio 1.4.8 - Nivel AAA)
  • Justificación del texto: El texto no está justificado para evitar problemas de espaciado entre palabras. (WCAG 2.2, Criterio 1.4.8 - Nivel AAA)
  • Foco: El indicador de foco está claramente visible para facilitar la navegación con el teclado. (WCAG 2.2, Criterio 2.4.7 - Nivel AA)
  • Enlaces "saltar a": Se han implementado enlaces "saltar a" para permitir a los usuarios del teclado acceder directamente al contenido principal. (WCAG 2.2, Criterio 2.4.1 - Nivel A)
  • Orientación: El diseño se adapta a diferentes orientaciones de pantalla (horizontal y vertical). (WCAG 2.2, Criterio 1.3.4 - Nivel AA)

Estilos Alternativos

El sitio web proporciona los siguientes estilos alternativos para adaptarse a las necesidades de los usuarios:

  • Modo noche: Un estilo con colores oscuros para reducir la fatiga visual.
  • Versión impresa: Un estilo optimizado para la impresión.
  • Bajo contraste: Un estilo con bajo contraste para usuarios con sensibilidad visual.
  • Tamaño de letra grande: Un estilo con un tamaño de letra más grande.

Estos estilos se aplican en el navegador (Firefox es el más adecuado para aplicarlos), en la pestaña "Ver" -> "Estilo de Página"
